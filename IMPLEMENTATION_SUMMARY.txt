KEY IMPLEMENTATION DETAILS:
================================================================================

State Prediction (lines 507-559 in mpc_agent.py):
  - LONGITUDINAL ONLY: Predicts velocity forward using physics model
  - LATERAL: Uses current state (x, y, psi, CTE, epsi) for immediate response
  - Physics model includes drag and rolling resistance
  - Handles num_steps = int(latency / dt) integration steps

Prediction Strategy (IMPORTANT):
  - Only velocity is predicted forward (longitudinal compensation)
  - Lateral states (x, y, psi, CTE, epsi) use current values
  - Rationale: Steering needs immediate response to stay on lane
  - Acceleration/braking can tolerate prediction for smoother control

Velocity Prediction:
  - Simple forward integration: v_next = v + a_net * dt
  - Accounts for drag: a_drag = 0.5 * Cd * rho_a * Av * v^2 / Mh
  - Accounts for rolling resistance: a_roll = mu * g
  - Net acceleration: a_net = a_control - a_drag - a_roll

Vehicle Parameters (synced with mpc_controller.py):
  - Lf = 2.0 m (wheelbase)
  - Cd = 0.3, rho_a = 1.225 kg/m³, Av = 2.2 m²
  - Mh = 1500 kg, mu = 0.01, g = 9.81 m/s²

================================================================================
TUNING RECOMMENDATIONS:
================================================================================

Default Setup (Works for most cases):
  - latency_sec: 0.2
  - enable_compensation: true
  - enable_adaptive: false

High Latency Networks (0.5-0.8s):
  - May need to reduce MPC horizon (N) or increase weights
  - Consider enabling adaptive latency
  - Monitor CTE values in logs

Local/Low Latency (< 0.2s):
  - Can disable compensation for simplicity
  - Or use lower latency_sec value

================================================================================
TESTING PROCEDURE:
================================================================================

1. Baseline (no compensation):
   - Set enable_compensation: false
   - Run: python3 main.py MPC
   - Save log: mv latency_compensation_log_*.csv baseline.csv

2. Compensated:
   - Set enable_compensation: true
   - Set latency_sec: 0.2 (or your target value)
   - Run: python3 main.py MPC
   - Save log: mv latency_compensation_log_*.csv compensated.csv

3. Validate:
   - python3 src/evaluation/validate_latency_compensation.py \
       --baseline baseline.csv \
       --compensated compensated.csv \
       --output ./validation_results

4. Review:
   - Check validation_report_*.txt for metrics
   - View plots: validation_comparison_*.png, latency_analysis_*.png
   - Verify CTE reduction and trajectory smoothness

================================================================================
TROUBLESHOOTING:
================================================================================

Vehicle deviates from lane:
  → Latency compensation is over-predicting
  → Reduce latency_sec value (try 0.1 or 0.15)
  → Disable compensation temporarily to verify

Vehicle still oscillates with compensation ON:
  → Under-compensation or network latency varies
  → Enable adaptive latency: enable_adaptive: true
  → Increase latency_sec slightly

Large CTE values in logs:
  → Check if path polynomial is valid
  → Verify waypoints are being tracked correctly
  → Review MPC weights (especially cte weight)
